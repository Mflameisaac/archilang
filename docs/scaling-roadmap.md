# archilang スケーリングロードマップ

> 2026-03-14 作成 — Codex (gpt-5.3) + Claude (opus-4.6) + Web調査の統合分析

## ポジショニング

CAD代替ではなく **「建築ルール付きDSLエンジン」** として差別化する。

勝ち筋は **企画初期の自動検証・案比較エンジン**、さらに将来的には **AIが生成した間取りを建築ルールで検証する中間表現** としてのポジション。

### 競合比較

| 比較対象 | 差別化ポイント | 弱点 |
|---|---|---|
| Mermaid / D2 | 建築ドメイン意味（mm座標・壁厚・到達性検証）がある | エコシステム・学習資産で圧倒的に不利 |
| JW-CAD / AutoCAD | 同一YAMLから同一図面の再現性、CI自動検証、大量パターン生成 | 作図自由度・注記・印刷・法対応で大きく不足 |
| AI間取り生成（まどりLABO, Maket.ai等） | 制約ベースで検証可能、決定論的 | AI生成は3分で5パターン出せる速度感に勝てない |

### 市場ニーズ

需要は「広く浅く」ではなく **「狭く深いB2Bニーズ」**。

| ターゲット | ニーズ |
|---|---|
| ハウスメーカー・規格住宅 | 反復プランを大量生成/検証 |
| デベロッパー企画部門 | ボリューム検討と面積試算の高速化 |
| 設計DX / PropTech | ルールベース生成のプロダクト化 |
| 計算設計志向の設計事務所 | Git/CIで設計品質管理 |

非エンジニア中心の設計チーム単体では、GUIなしのテキスト専用運用は普及しにくい。

## 必要機能の優先順位

### Tier 1: ツールとして成立するための最低条件

| # | 機能 | 理由 |
|---|---|---|
| 1 | **多層階対応** | 住宅の99%は2階建て以上。1F固定では木造住宅すら扱えない |
| 2 | **階段・EV表現** | 上下階接続のトポロジーがないと多層階の意味がない |
| 3 | **面積表（法的面積計算）** | 延床・容積対象・建築面積の自動算出。確認申請の初期チェックに直結 |
| 4 | **DXF出力** | 既存CADワークフロー（特にJW-CAD）への受け渡し必須。IFCより先 |

### Tier 2: 実務PoC（プロトタイプ検証）に必要

| # | 機能 | 理由 |
|---|---|---|
| 5 | **設備配置（キッチン/UB/トイレ）** | 水回り配置なしの間取りは「部屋割り図」であって「間取り」ではない |
| 6 | **構造グリッド整合チェック** | 柱割り逸脱の自動警告。木造軸組ならモジュール整合は最低限 |
| 7 | **テンプレート/プリセット** | 3LDK/4LDK等の初期案を数秒で生成。手書きYAMLの障壁を下げる |
| ~~8~~ | ~~**multi-rect + sub_rooms同時対応**~~ | ✅ **実装済み (v0.3)** — cellBasedSplit + flood-fill対応 |

### Tier 3: プロダクト化に必要

| # | 機能 | 理由 |
|---|---|---|
| 9 | **GUI（YAML↔ビジュアル双方向同期）** | 非エンジニア設計者の採用障壁を下げる唯一の手段 |
| 10 | **法規チェック（採光/換気/避難経路）** | DSLエンジンの最大の差別化ポイント。ルールベースで自動検証 |
| 11 | **IFC出力** | BIM連携。buildingSMART IFC標準準拠 |
| 12 | **LLM連携（自然言語→YAML生成）** | AI間取りツールに対抗する「テキスト→YAML→検証→SVG」パイプライン |

## 技術的ボトルネック（現設計の阻害要因）

| 問題 | 箇所 | 影響 |
|---|---|---|
| 1F固定のresolve | `resolver.ts:24` | 多層階拡張にデータモデル再設計が必要 |
| 直交壁限定 | `resolver.ts:445` | 斜め壁・R壁が永遠に不可 |
| ~~multi-rect + sub_rooms排他~~ | ~~`resolver.ts:851`~~ | ✅ **解消済み** — 完全分断(flood-fill) + 部分壁(cellBasedSplit) 両対応 |
| 壁のrooms属性が空 | `resolver.ts:470` | 明示壁が部屋との紐付けを持たず、後段バリデーションが不安定 |
| 整数mmパーサ | `parser.ts:93` | `1820.5mm`等の実務値に対応不可 |
| 開口部干渉チェックなし | `resolver.ts:530` | 建具同士の重複配置を検出できない |
| BFS接続性検証のみ | `validator.ts:265` | 動線幅・避難・法規の実務条件を表現できない |
